Where the parser breaks · A PM problem

Mood is universal.
The audio of mood isn't.

When I started, I assumed "workout" meant the same audio features everywhere. The original analysis project I built — a country comparison of South Korea, USA, and Mexico using t-tests on Spotify's top 50 playlists — proved that assumption wrong. Korean top tracks had measurably lower energy than US ones. Mexican tracks had higher danceability than both. The mood is the same; what fits it changes by region.

You can see the same pattern at the genre level in this dataset. Latin-American genres (reggaeton, tropical, latin pop) sit at the high-valence high-danceability corner. EDM is high-energy but emotionally cold. Rock is the opposite of dance pop on every axis except tempo. A vibe-to-feature mapping that doesn't know this returns "American workout music" to everyone who types "workout."

Product takeaway

v1.0 should geo-calibrate the parser. The same prompt ("workout") should produce different feature targets for different users — biased toward locally-popular sub-genres, not Western pop defaults. This isn't a translation problem; it's a cultural calibration problem. The data needed to do it well already exists in any music platform's regional charts.

How this works.

v0.1 · deterministic engine Demo mode

Step 1 · Intent parsing. The user's free-text vibe is parsed into target audio-feature ranges — energy, valence, danceability, tempo, acousticness — plus weighted genre preferences. This demo ships with a deterministic keyword-weighted parser so it runs offline with zero cost. The production version swaps in an LLM call (Claude Sonnet) — code preserved in parseVibeAI(), behind a feature flag.

Step 2 · Retrieval. Each of 2,048 candidate tracks is scored by weighted Euclidean distance to the target vector. Genre weights act as multipliers; popularity is a tiebreaker. Top 10 returned, with diversity enforced (max 2 songs per artist).

Step 3 · Explanation layer. A short rationale connects the user's words to the selected songs. Templated in demo mode; LLM-generated in production. This is the layer most recommenders skip — and the one users care about most.

Why ship deterministic v0.1 first? The riskiest assumption isn't "can an LLM parse mood" — it's "do users actually prefer mood-based discovery over genre-based." A deterministic parser is good enough to test that, runs for free, and never fails in a demo. The LLM upgrade adds quality on edge cases (compositional prompts, intensity nuance) once the core product hypothesis is validated.
